Run 5's TL forgot `replan_reason`. The validator caught it (good), but ideally TL gets it right on the first iter-2 emit. Two paths:

1. **Stronger prompt nudge** — make `replan_reason` a top-level required field in the iter-2 emit schema, not just a soft "you must include." About 15 min of prompt work.
2. **TL self-critique check** — add a c9 that validates `replan_reason` presence when iteration > 1 IN-LOOP, so TL sees the validation failure before emit and corrects in-turn. Engineer never receives a bad plan; commander never has to re-dispatch TL.

Either keeps the SHIP rate higher on 2-iter runs. Not a launch blocker — the validator catches it; we just spend an extra TL re-dispatch cycle when it forgets.
