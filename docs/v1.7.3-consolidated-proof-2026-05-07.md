## 1. What v1.7.3 set out to achieve

The v1.7.3-ledger MVP scope (from
[`docs/v1.7.3-ledger-mvp-2026-05-06.md`](v1.7.3-ledger-mvp-2026-05-06.md)):

> Manually run shadow mode on selected GitHub workflow failures and
> store evidence in a ledger. Engineer must NOT push in shadow_mode.

The architectural goal: prove that Phalanx can observe real CI failures
on third-party repos, produce diagnostic output, AND refuse to ship
unsafe code — all without touching the repo. The ledger captures
TL's verdict + diagnosis + cost + time per attempt; aggregate metrics
distinguish "architecture safety wins" (SAFE_ESCALATE) from
"infrastructure failures" (FAILED + failure_class).

Hard constraints throughout:
- Third-party repos only (no synthetic reverts beyond the initial
  sanity-check Path A on usephalanx forks)
- Shadow_mode true → engineer must not push
- Zero git mutations on monitored repos
- Verdict semantics must distinguish architecture refusal (SAFE_ESCALATE)
  from infra defects (FAILED + failure_class)

## 8. Three remaining FAILED entries — honest read

The latest-per-workflow ledger has 3 FAILED entries. None weakens
the v1.7.3 thesis:

### 1. usephalanx/humanize Path 2 (Challenger Celery hang)

Pre-hardening run from before the v1.7.3 stuck-task detector existed.
TL emitted at confidence 0.95 with a precise root_cause
(`intword` locale decimal separator regression). The Challenger
agent then hung in Celery for 25+ min until the runner's poll
timeout. **Architecturally, the run produced perfect TL output
preserved in the ledger**; only the orchestration runtime hung.
Re-running today under v1.7.3 hardening would land cleanly. Counted
as FAILED for accuracy of the historical record.

### 2. sphinx-doc/sphinx (uv `--group` syntax)

`uv pip install -r pyproject.toml --group package --group test
--group types` fails in the sandbox. Sphinx uses PEP 735
`[dependency-groups]` which requires uv ≥ 0.5.x; the installed uv
in the sandbox image is older. Fix is either:
- Pin a newer uv install in `_handle_setup_uv`
- Detect PEP 735 deps and fall back to `pip install -e .[group]` syntax

Both are scoped extractor refinements; deferred until a second repo
exhibits the same shape.

### 3. aio-libs/aiohttp (Cython transpilation)

After 4 generalizable fixes (NM1-NM4), the deepest remaining failure
is aiohttp's setup.py expecting `mask.c` to exist on disk during
`build_ext`, but `mask.c` is generated from `mask.pyx` by Cython —
which apparently runs in a separate phase aiohttp's build doesn't
trigger in our env. This is package-specific; not a pattern.

  hardening doc) and 3.3 min average per shadow run.

The platform behavior is the architectural property we wanted to
prove: **refusing to ship in unclear situations rather than
guessing**. Every SAFE_ESCALATE is the architecture working
correctly. Every retry is the system surfacing the next real
problem rather than blocking on noise. The ledger is the proof.
